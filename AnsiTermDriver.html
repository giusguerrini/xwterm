Class: AnsiTermDriver

AnsiTermDriver(params)

This is the base class from which AnsiTerm's communication drivers inherit the interface. Internal drivers AnsiTermHttpDriver and AnsiTermWebSocketDriver are derivations of this class. The programmer can write their own driver to connect the terminal to the backend that implements the specific application. AnsiTerm uses public methods of this interface to communicate with the backend. Programmers who need to write a custom driver are not required to implement all the methods provided by this interface. In particular, it is unlikely that it is ever necessary to rewrite the callback registration methods, since the base class already implements a sufficiently general mechanism. A typical driver will redefine these methods: _tx _set_connection_state start _stop close setSize
